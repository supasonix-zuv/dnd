Obsolete UA
Fredbob711 23 Nov 2020 20:59

So I'm curious what everyone thinks about obsolete UA. With Tasha's coming out, there's a significant number of UA subclasses that are no longer valid because they've been promoted to published content. We have people adding that content now (thank you everyone who is contributing). I can think of a couple options for what we can do.

  1. Retain Obsolete UA Content
    1. Main Page
      1. A: Leave Obsolete UA where it is
      2. B: Move Obsolete UA to a separate section under each class heading
      3. C: Move Obsolete UA to a separate section entirely, similar to the Homebrew content
    2. Class Pages
      1. A: Leave Obsolete UA where it is
      2. B: Move Obsolete UA to a separate section similar to other homebrew content
  2. Remove Obsolete UA content
    1. Main Page: Self Explanatory
    2. Class Pages: Self Explanatory
      1. An alternative to removing the data entirely is to add references and/or links to the UA content on the published subclass pages. This would remove the clutter of having obsolete UA, but would also keep it easily accessible for anyone that wants to see it.

I'm leaning towards option 2, just removing the UA content entirely, but adding in the references on the published subclass pages.

This also brings up old UA that was never published, like the City & Protection Domains for the cleric. That can be addressed later, there's a lot less of those than things that became obsolete because of the release of Tasha's.

Re: Obsolete UA
qorinthian 23 Nov 2020 22:53

I agree with Option 2. The problem with UA is when revisions of the same UA comes out, it starts to clutter the main and class pages. For class pages, I'd even advocate for having a section for Published content, then a second combined section with UA and Homebrew content. (I personally think homebrew content not by Wizards is already questionable, but it's okay.)

This also extends to the Ranger. Now that Tasha's has provided alternate class options to make the Ranger better, I really think we should revert Ranger to PHB, and create a new page for Revised Ranger. PHB Ranger can link to published subclasses; Revised Ranger can link to UA versions of subclasses.

cottoneyenoh 24 Nov 2020 05:12

I think that having a link on the page leading to obsolete UA might be nice so that we preserve the old versions of them, but getting them off of the main page to reduce clutter would be helpful. Additionally, separating the published content from the remaining UA so that it isn't all one amorphous blob on the main page would be very nice as well.

I also agree regarding the ranger. We're working on getting the PHB ranger with the optional class features from Tasha's, but it'll take some time, especially if we want to keep the revised ranger options on the website rather than erase them entirely.
